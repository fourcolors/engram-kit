# PageIndex vs Observational Memory — head-to-head

9 document-content probes (a specific detail buried in an academic paper, with a
gold answer + page refs, generated by reading the actual papers). Each answered
two ways, judged against the paper:
- **OFF** = observational memory only (agent-memory, PDF reading disabled)
- **ON**  = PageIndex (PDF reading enabled)

## Scoreboard (the contested surface: document-content questions)

| Metric | OFF (memory only) | ON (PageIndex) |
|---|---|---|
| Avg score / 5 | **1.00** | **4.56** |
| Win / tie / loss (ON vs OFF) | — | **8 / 1 / 0** |
| Abstained | **9 / 9** | 1 / 9 |
| Correct | 0 / 9 | **8 / 9** (7×5, 1×4) |
| Wrong / hallucinated | 0 | **0** |

**PageIndex wins by +3.56/5.** OFF abstained on every probe — honest ("the
workspace files don't contain this"), zero hallucinations, but zero answers. ON
read the paper and gave verbatim, page-cited facts (164 LongMemEval attributes;
LifeBench 58h→8h; MT-Bench 65.0%→77.5% / 4× cost; G-Eval's LLM-summary bias;
Arena-Hard BT-MLE 3× weighting / R²=0.998; FActScore $4/gen / Upwork $15–25/hr;
FACTS +3.23% self-preference / Condorcet; Let's-Verify 80/20 split / 2.6× efficiency).

## The verdict: NOT a competition — they cover different surfaces

| Surface | Best system | Score |
|---|---|---|
| **Evolving workspace** (the 10 sample Qs: temporal, no-hindsight, supersession) | **Observational memory (v4)** | 4.60/5 — PageIndex inert on 9/10 |
| **Static document contents** (these 9 probes) | **PageIndex** | 4.56/5 vs memory's 1.00/5 |

They are near-orthogonal: on workspace questions PageIndex is silent (nothing to
read); on document questions observational memory abstains (never ingested the
paper bodies, correctly refuses to invent). **Each rescues the cases the other
cannot reach.** "Better or worse" only has an answer per-surface.

**Recommendation:** keep PDF reading **opt-in (default off)**; turn it on for
document-heavy question sets, off for pure workspace/temporal questions. Route by
question type — don't substitute one for the other.

## PageIndex failure modes exposed (both conservative — no hallucinations)
1. **Wrong-section navigation** (`probe_2026_he_memoryarena`): the nav landed in
   Experiments (pp.10-12) and never reached the task-construction text on p.4, so
   ON abstained (tie). The fact was in the document; the tree didn't route to it.
   *Fix:* if the selected nodes don't contain the asked detail, fall through to a
   broader/sibling-node scan before abstaining; index method/dataset-construction
   sections explicitly (common home for "how many / how built" facts).
2. **Over-hedging on a verbatim fact** (`probe_2023_min_factscore`): called the
   $4/generation figure "implied" though p.4 states it outright (scored 4 not 5).
   *Fix:* when the exact string is retrieved, cite it as stated, not hedged.

Both failure modes fail *safe* (abstain / under-claim), never dangerous.
