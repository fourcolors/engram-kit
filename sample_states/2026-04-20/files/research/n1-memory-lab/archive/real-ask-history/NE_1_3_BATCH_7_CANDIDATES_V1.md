# NE-1.3 Batch 7 Candidates V1

This is the seventh discovery batch of `5` asks.

These extend the set into restart-path quality, canonical artifact choice,
trust-first routing, project-set recovery, and durable learning transfer.

## Candidate 1

**Ask**

`what should I open first if I want to resume this current raw-first bundle work?`

**Why include**

- strongest restart-path ask
- tests whether the system can route to the right re-entry artifact instead of producing another recap

## Candidate 2

**Ask**

`what is the canonical restart artifact for this period, and why is it that object rather than a summary dump?`

**Why include**

- strongest canonical-artifact choice ask
- tests whether the system can identify the right continuity object, not just any helpful summary

## Candidate 3

**Ask**

`which memory layer should be trusted first in this bundle, and what should it be trusted for?`

**Why include**

- strongest trust-first / search-entropy reduction ask
- tests whether the system can choose among raw traces, handoffs, carry-forward objects, and code-state anchors cleanly

## Candidate 4

**Ask**

`what projects am I working on?`

**Why include**

- strong project-set recovery ask
- prevents the set from collapsing into only one-thread continuity
- useful for evaluating broader working-model reconstruction

## Candidate 5

**Ask**

`what have we learned from actual usage about how data flows across platforms?`

**Why include**

- strong durable learning transfer ask
- genuinely cross-harness
- tests whether the system preserves lessons that should survive beyond one session or one implementation state

Batch 7 adds a strong practical-value layer:

- restart routing
