# Environment Wording Notes — 2026-04-19

Working wording for the replay-native environment definition.

## Tight formulation

**One human, many harnesses, many partial memory surfaces, and one memory process responsible for maintaining a coherent and useful externalized self/world model across them over time.**

Why this is better than simpler "one human, many harnesses, one memory process":

- it preserves the fact that other memory surfaces already exist
- it makes the fragmentation/federation problem explicit
- it explains why the task is interesting without sounding benchmarky
- it keeps the role of Syke specific: not the only memory, but the process
  trying to maintain coherence across partial memories

## Task implication

At time `t`, using only the evidence available by then, the system should
maintain and expose that externalized self/world model in a way that supports
useful continuation.

This is a cleaner center than:

- retrieval
- summary
- reconstruction alone

because it implies maintenance, update, exposure to asks, and continuity over
time without forcing explicit jargon about partial observability.
