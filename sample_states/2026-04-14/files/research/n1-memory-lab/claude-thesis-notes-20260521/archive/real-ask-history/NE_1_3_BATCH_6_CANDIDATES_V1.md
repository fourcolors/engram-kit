# NE-1.3 Batch 6 Candidates V1

This is the sixth discovery batch of `5` asks.

These close the most important remaining gaps after batches 1-5:

- stale-memory verification
- restart-path / canonical artifact choice
- clean-baseline truth
- stale-story rejection
- explicit environment-change truth

## Candidate 1

**Ask**

`what's the current state of things, and what in memory is stale or no longer true?`

**Why include**

- strongest stale-memory verification ask
- tests correction against current reality, not just recap
- should likely recur across time in the eventual `50`

## Candidate 2

**Ask**

`what should I open first if I want to resume this current raw-first bundle work?`

**Why include**

- strongest restart-path / first-read routing ask
- tests whether the system can reduce search entropy immediately
- useful counterpart to handoff / last-session recovery

## Candidate 3

**Ask**

`is the replay or sandbox baseline actually clean yet, or are we still contaminated?`

**Why include**

- strongest clean-baseline truth ask
- asks whether the experiment itself is trustworthy
- very high practical value

## Candidate 4

**Ask**

`what old architecture story is no longer true now?`

**Why include**

- strongest stale-story rejection ask
- tests whether the system can kill obsolete mental models instead of repeating them
- especially important in later NE-1.3

## Candidate 5

**Ask**

`what changed now that codex is part of the object?`

**Why include**

- strongest explicit environment-change ask
- makes harness-mix drift visible instead of implicit
- good repeated-shape / before-vs-after candidate

## Batch 6 role

Batch 6 adds the remaining high-value truth surfaces:

- stale memory against current reality
- restart artifact selection
- experiment honesty
- stale-story rejection
- explicit before/after environment change
