# NE-1.3 Recurrence Priority V1

These are the ask shapes that should recur multiple times in the eventual `50`.

The point is not redundancy.
The point is to capture that the same ask can have a different right answer as
the month and harness mix evolve.

## Highest-priority repeated shapes

1. **Current state / where are we now**
2. **What changed**
3. **What is implemented now**
4. **What is still live vs residue**
5. **What happened in the last session / what was I last working on**
6. **Why does X still feel incomplete**
7. **Current runtime/setup truth**

## Why repeat them

- before vs after Codex materially enters
- before vs after the runtime/ontology shift
- burst-era versus later-month state
- thread-recovery versus code-state truth

## Practical use

When filling the set toward `50`, prefer duplicating these ask shapes across
different times over inventing more one-off prompts.
