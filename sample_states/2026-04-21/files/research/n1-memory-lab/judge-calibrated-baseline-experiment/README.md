# Judge-Calibrated Baseline Experiment

Status: active experiment space, created 2026-04-21.

Purpose: turn the judge-calibration paper sweep and the current Syke replay
packet into a disciplined measurement program. This is not an architecture
leaderboard yet. It is an experiment to validate the measuring instrument
before trusting any architecture ranking.

## Core Frame

The observed verdict is not ground truth.

```text
Y = Judge(r_t, q_t, E_t, rubric, model, scale, prompt_order, stochasticity)
```

The benchmark unit is:

```text
I_t = (t, q_t, E_t, r_t)
```

Where:

- `t` is the reference time.
- `q_t` is the ask.
- `E_t` is the bounded evidence packet.
- `r_t` is the answer or reconstruction.

The experiment asks:

```text
Can this judge setup observe Syke-relevant quality without obvious scale,
packet, rubric, model, or verifier artifacts?
```

It does not yet ask:

```text
Which Syke architecture wins?
```

## Important Corrections

- `zero` is a valid bounded-packet condition. It is not a no-tool or
  no-retrieval condition.
- `pass`, `partial`, and `fail` are noisy ordinal observations generated by a
  linguistic judge process. They are not truth labels.
- Consensus is stability evidence, not correctness evidence.
- More rubric detail is not automatically better.
- The current packet is mainly useful for judge/rubric calibration, not mature
  architecture ranking.

## Work Rule

Do real science first:

1. Define the object.
2. Freeze the packet schema.
3. Validate the judge/rubric/scale.
4. Measure reliability.
5. Only then compare architectures.
