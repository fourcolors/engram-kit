# NE-1.3 Batch 3 Candidates V1

This is the third discovery batch of `5` real asks.

These extend the set into the later month where the continuity burden shifts
toward architecture/state truth, synthesis/runtime truth, and replay/sandbox
truth.

## Candidate 1

**Ask**

`what is happening with synthesis?`

**Why include**

- strongest direct synthesis-state ask
- good later-month counterpart to earlier continuity-only asks
- important for truthfulness when the architecture changes quickly

## Candidate 2

**Ask**

`what experiments are being done with observe?`

**Why include**

- strong replay/eval correctness ask
- captures the shift from Observe as architecture thread to Observe as active testing surface

## Candidate 3

**Ask**

`what sandbox is running?`

**Why include**

- strong contained-state truth ask
- excellent for later-month truthfulness because sandbox/replay mechanics changed quickly

## Candidate 4

**Ask**

`what is the current state of each adapter?`

**Why include**

- strong architecture/runtime truth ask
- forces distinction between design intent, landed implementation, and still-open work

## Candidate 5

**Ask**

`why might the user feel that Observe is still not complete?`

**Why include**

- strong repeated-shape ask whose correct answer changes over time
- good bridge between continuity and code-state truth

## Batch 3 role

Batch 1 core `6` covers immediate continuity burden.

Batch 2 adds unfinished-work, project-state, durable learning, and
cross-platform understanding.

Batch 3 adds:

- synthesis truth
- replay/sandbox truth
- adapter/runtime truth
- evolving incompleteness / gap truth

That moves the set toward a real month-scale `NE-1.3` object rather than a
single burst snapshot.
