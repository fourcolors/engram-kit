# Ask Pressure Matrix — NE-1.3 15-Day Slice

This is the working concept-by-ask matrix for the first 15-day slice
(`R01-R24`, dated `2026-03-08` through `2026-03-22`).

It is not the final ontology.
It is a practical working matrix for benchmark scope and judge redesign.

## Pressure set used

- `operative_state`
  current/active/latest/open/unfinished/next-step recovery

- `bounded_history`
  yesterday/today/last-session/last-week/date-bounded reconstruction

- `object_continuity`
  evolving named object, design, subsystem, or issue across time

- `completeness`
  asks that explicitly demand exhaustive/thorough/list-everything/countable coverage

- `cross_surface`
  asks that materially require combining more than one harness/surface

- `committed_truth`
  asks whose correct answer depends on code/config/docs/runtime truth, not just discussion memory

- `provenance_audit`
  asks that explicitly demand how the answer was obtained or derived

## Why this matrix exists

The point is not to freeze categories.
The point is to make the current 15-day benchmark surface auditable.

This helps answer:

1. what this ask set is actually stressing
2. which judge obligations are universal
3. which judge obligations should only fire conditionally

The 15-day slice starts with:

and gradually broadens into:
